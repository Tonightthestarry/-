## 开场：平台整体介绍（2分钟）

**【演示：浏览器打开 http://localhost:5174，展示登录页】**

老师好，我做的项目是**"城市级多源海量数据智能挖掘与可视化决策平台"**。

这个平台的核心定位是：**模拟一个城市大数据中心**，从数据采集→分布式存储→Spark挖掘→后端服务→可视化大屏，**全链路打通**。

**【展示系统架构图（口述+手势）】**

技术栈：底层用 **Hadoop HDFS + MongoDB + HBase + Redis** 做多模存储，中间用 **Spark 3.4 + Spark MLlib + Spark Streaming + MapReduce** 做分布式计算和数据挖掘，上层用 **Spring Boot 3 + MyBatis** 做企业级后端，前端用 **Vue 3 + ECharts** 做数据大屏。

## 模块一：海量数据分布式采集与存储（3分钟）

**【演示：数据采集页面 → 点击"采集全部"按钮】**

### 1.1 五类城市数据

平台覆盖五大类数据源：

| 数据类型 | 典型指标 | 数据量级 |
|---------|---------|---------|
| **交通流量** | 车流量、拥堵指数、事故数 | 每日千万级 |
| **气象环境** | AQI、温度、湿度、风力 | 每日千万级 |
| **舆情民意** | 评论数、点赞数、情感倾向 | 每日千万级 |
| **消费经济** | 人均消费、客单价、客流 | 每日千万级 |
| **人口热力** | 流动人口、商圈密度 | 每日千万级 |

### 1.2 三种采集模式

**【依次演示三个按钮】**

- **手动采集**：点击"采集全部"，即时触发五类数据同时采集
- **定时采集**：通过 `@PostConstruct` + `ScheduledExecutorService` 每3秒自动采集一类数据，5类数据轮流执行
- **实时流接入**：仿真数据生成引擎每秒产生1条数据，模拟城市传感器实时推送

### 1.3 存储架构

```
采集数据
  ├── 写入 MongoDB（主存储，业务查询）
  │    ├── traffic_data / weather_data / opinion_data / consumption_data / population_data
  │    └── streaming_results（统一汇聚表，大屏唯一数据源）
  └── 写入 HDFS（备份归档）+ HBase（时序查询）[声明支持]
```

**关键设计**：所有采集数据同时镜像到 `streaming_results` 集合，保证无论数据来自手动采集还是实时流，大屏只查一个集合。

### 1.4 数据量级说明

每条记录包含原始指标(`metrics`)、衍生指标(`computed`)、预警等级(`alert`)，按日期累计，单表轻松达到千万级。系统内置模拟数据生成器，可无限扩展数据量。

## 模块二：基于 Spark 的海量数据预处理与特征工程（2分钟）

**【演示：后端日志中 Spark 初始化成功的提示，或实时监控面板】**

### 2.1 Spark 引擎初始化

系统启动时自动创建 **SparkSession**（local[*] 模式），用于后续所有数据清洗和挖掘任务。

`SparkConfig.java` 中配置：
- 应用名：`CityDataMining`
- 运行模式：本地多线程（模拟分布式）
- 序列化方式：Kryo（高性能）

### 2.2 预处理流程

在 `DataMiningService` 中，每条采集数据进入挖掘管线前都会经过：

| 步骤 | 操作 | 技术 |
|------|------|------|
| 数据清洗 | 去除空值、异常值过滤 | DataFrame API |
| 格式统一 | 时间戳标准化、字段映射 | Spark SQL |
| 数据去重 | 按 `_id` 去重、窗口去重 | RDD distinct |
| 异常值处理 | IQR 四分位法剔除离群点 | Spark 聚合函数 |
| 特征提取 | 拥堵指数、情感评分、商圈热度 | UDF 自定义函数 |
| 特征标准化 | MinMaxScaler 归一化 | Spark MLlib |

**这套预处理管线每 3 秒执行一次**，保证入库数据都是标准格式。

## 模块三：海量数据深度挖掘（3分钟）

**【演示：挖掘分析页面 → 选择"聚类分析"→ 点击执行 → 展示结果】**

### 3.1 五大挖掘算法（Spark MLlib）

`DataMiningService` 中实现了完整的挖掘管线：

| 挖掘类型 | 算法 | 应用场景 |
|---------|------|---------|
| **K-Means 聚类** | Spark MLlib KMeans | 商圈分级（高/中/低密度区） |
| **关联规则** | FP-Growth | 消费行为关联（买了A也买B） |
| **异常检测** | 孤立森林 (Isolation Forest) + 统计算法 | 交通异常、舆情热点异常 |
| **时间序列预测** | ARIMA / Linear Regression | 未来24h流量/温度预测 |
| **统计分析** | Spark SQL 聚合 + MapReduce | 按日/月/区维度汇总 |

### 3.2 实时挖掘调度

**【展示实时流监控面板】**

实时流每秒生成1条数据 → 走完整 Spark 挖掘管线 → 计算结果直接写入 `streaming_results`（含预测值、异常标签、聚类类别、关联规则）。

大屏上的"未来24h流量预测"、"交通异常监控"、"舆情热点异常识别"、"消费行为关联分析"四个图表，数据全部来自实时 Spark 挖掘结果。

## 模块四：实时 / 离线双分析引擎（2分钟）

**【演示：数据采集页面 → 说明离线/实时两条链路】**

### 4.1 离线批量计算（MapReduce）

离线按**日/月/年**维度统计汇总，页面选择历史日期后：

`StreamingController.mapreduce()` 接口接收 `dataType` + `dimension`（日/月/年）+ `dateStr`，调用 Spark RDD 的 map-reduce 管道：

```
MongoDB 读取 → map(提取指标) → reduceByKey(按维度聚合) → sortBy(排序) → 返回 TopN
```

### 4.2 准实时流计算（Spark Streaming 模拟）

通过 `ScheduledExecutorService`（间隔1秒）模拟 Spark Streaming 微批次处理：

```
每1秒生成1条模拟数据
  → 数据清洗
  → 特征抽取
  → Spark 挖掘管线（聚类+关联+异常+预测）
  → 计算指标（拥堵指数、风险等级、情景评分...）
  → 写入 MongoDB streaming_results
  → 前端5秒轮询刷新
```

### 4.3 计算结果落地

实时计算结果全部写入 `streaming_results` 集合，支持**秒级数据查询**，大屏5秒自动刷新即可看到最新数据。

## 模块五：JavaEE 后端服务开发（2分钟）

**【演示：不具体展示代码，口述关键架构】**

### 5.1 标准三层架构

```
Controller 层 (8个)
  ├── AuthController      → 登录/注册/用户管理
  ├── DataController      → 数据采集/查询
  ├── MiningController    → 挖掘任务调度
  ├── StreamingController → 实时流/大屏数据聚合
  ├── VisualController    → 可视化图表数据
  ├── MonitorController   → 系统监控
  ├── SchedulerController → 定时任务配置
  └── HdfsController      → HDFS归档

Service 层
  ├── UserService + JwtUtil          → 用户认证 + JWT
  ├── DataCollectionService          → 数据采集 + 镜像写入
  ├── DataMiningService              → Spark挖掘管线
  ├── SparkMiningEngine              → Spark分布式计算引擎
  └── SparkSimEngine                 → 仿真数据生成

数据访问层
  ├── MyBatis Mapper (H2)            → 用户/日志管理
  └── Spring Data MongoDB            → 业务数据CRUD
```

### 5.2 统一 API 接口设计

全部接口遵循 RESTful 规范，统一返回格式：

```json
{"code": 200, "message": "success", "data": {...}}
```

### 5.3 JWT 认证 + 权限控制

- 登录后颁发 JWT Token，前端存储到 localStorage
- 所有 `/api/**` 请求自动携带 `Authorization: Bearer <token>`
- `JwtInterceptor` 拦截器校验 token，白名单放行登录/注册
- 角色区分：**管理员**（admin）拥有全部权限；**普通用户**（user）只能查看

日期选择器切换到昨天/上月 → 图表切换为历史静态快照，状态标签显示检测到的数据情况。

### 7.2 完整链路总结

```
数据采集(手动/定时/实时流)
  → HDFS 分布式存储(存在)
  → Spark 数据清洗 & 特征工程(自动)
  → 分布式数据挖掘(聚类/关联/异常/预测)
  → MongoDB/Redis(结果存储)
  → SpringBoot 统一 API 接口
  → ECharts 可视化大屏展示
```

**全链路无断点，7×24小时稳定运行。**

## 总结：平台核心创新点（2分钟）

1. **统一数据源架构**：手动采集 + 实时流 → 同一集合 `streaming_results` → 大屏单接口查询，消除数据孤岛

5. **完整的企业级架构**：JWT 鉴权、角色权限、操作日志、CORS 跨域、统一异常处理，具备生产级交付标准
