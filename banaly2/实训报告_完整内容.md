**基于多技术栈的B站数据分析与爆款预测系统**

数据科学与大数据技术专业  张三

【摘 要】随着短视频平台的快速发展，B站（哔哩哔哩）已成为国内重要的视频内容平台，每天产生海量的视频数据。如何从这些数据中挖掘有价值的信息，识别爆款视频的规律，对于内容创作者和运营人员具有重要的参考意义。本系统基于Python语言，整合Requests爬虫框架、Spark分布式计算引擎、MySQL和MongoDB双库存储、Plotly可视化图表等技术，设计并实现了B站视频数据分析与爆款预测系统。系统通过B站API接口采集视频数据，经数据清洗后按日期分表存入MySQL和MongoDB，利用Spark RDD实现基于标签分类的动态聚类分析，将视频按标签分类为"热门标签""中等标签""冷门标签"三类，分别采用不同的评判标准进行聚类（爆款/热门/普通/冷门），最后通过Streamlit框架构建Web应用界面，提供数据采集、数据概览、可视化图表和关联分析等核心功能。

【关键词】B站；数据分析；Spark；聚类分析；爆款预测；Streamlit


**Multi-technology Stack Based Bilibili Data Analysis and Hot Video Prediction System**

Data Science and Big Data Technology  ZHANG San

Abstract: With the rapid development of short video platforms, Bilibili has become an important video content platform in China, generating massive amounts of video data daily. Mining valuable information from this data and identifying the patterns of hot videos is of great reference significance for content creators and operators. This system, based on Python, integrates Requests crawler framework, Spark distributed computing engine, MySQL and MongoDB dual-database storage, and Plotly visualization charts to design and implement a Bilibili video data analysis and hot video prediction system. The system collects video data through Bilibili's API interfaces, cleans the data and stores it in MySQL and MongoDB by date-partitioned tables, utilizes Spark RDD to implement dynamic clustering analysis based on tag classification, classifies videos into "hot tags", "medium tags" and "cold tags" categories with different evaluation criteria for clustering (hot/trending/normal/cold), and finally builds a web application interface through the Streamlit framework, providing core functions such as data collection, data overview, visualization charts, and association analysis.

Key words: Bilibili; Data Analysis; Spark; Clustering Analysis; Hot Video Prediction; Streamlit


## 1 绪论

### 1.1 研究的背景和意义

B站（哔哩哔哩弹幕视频网）作为国内领先的年轻人文化社区和视频平台，已经发展成为集视频、直播、游戏、电商等多元业务于一体的综合性平台。截至2025年，B站月均活跃用户超过3.5亿，日均视频播放量超过40亿次。在如此海量的视频数据中，如何识别和分析爆款视频的特征规律，帮助内容创作者把握热点方向，成为数据分析领域的重要课题。

传统的视频分析往往采用单一的评判标准，即仅根据播放量或点赞数来判断视频的"热度"。然而，不同领域的视频天然存在流量差异——游戏、美食等热门分区的视频动辄数十万播放量，而手工、园艺等冷门分区的视频能达到几万播放量已经属于该领域的"爆款"。因此，亟需一种能够根据不同标签分类、动态调整评判标准的智能分析系统。

本课题旨在设计并实现一个基于多技术栈的B站数据采集、存储、分析与可视化系统。系统通过B站API接口自动采集视频数据，采用Spark RDD分布式计算引擎实现基于标签分类的动态聚类分析，最终通过Streamlit Web框架提供友好的用户交互界面，帮助用户快速了解视频热度分布、UP主排名、聚类类别等关键信息，为内容创作决策提供数据支持。

## 2 相关技术和工具

本系统的开发涉及多种技术栈，具体如下：

**（1）Python编程语言**
Python是系统的主要开发语言，版本为3.9+。Python具有丰富的第三方库生态，非常适合数据分析与Web开发领域。

**（2）Requests 网络请求库**
Requests是Python最流行的HTTP库，用于向B站API接口发送请求并获取JSON格式的响应数据。相比于BeautifulSoup，Requests更适合API接口的JSON数据交互。

**（3）PySpark / Spark RDD**
Apache Spark是一个快速、通用的分布式计算引擎。本系统使用PySpark在本地模式下运行（master="local[*]"），核心使用RDD（弹性分布式数据集）的parallelize和map操作实现并行聚类分析。RDD是Spark最基础的数据结构，支持分布式并行处理。

**（4）MySQL 关系型数据库**
MySQL用于存储结构化的视频数据。本系统采用按日期分表的策略（表名格式：bilibili_videos_YYYYMMDD），有效解决了单一表数据量过大的问题。使用pymysql库进行数据操作。

**（5）MongoDB 文档数据库**
MongoDB作为辅助存储，以文档形式保存视频数据，提供了数据备份和灵活查询的能力。使用pymongo库进行操作。

**（6）DataCleaner 数据清洗模块**
基于Pandas实现数据清洗，包括HTML标签移除、数字标准化（处理"万""亿"等单位）、去重、缺失值填充等操作。

**（7）Streamlit Web框架**
Streamlit是Python生态中的轻量级Web应用框架，能够快速将数据脚本转化为交互式Web应用，无需编写HTML/CSS/JavaScript前端代码。

**（8）Plotly 可视化库**
Plotly Express提供交互式图表生成能力，支持柱状图、饼图、气泡图等多种图表类型。

**（9）定时调度模块**
基于Python内置的threading模块实现后台定时采集功能，通过轮询检查时间触发每天定时自动采集任务。

## 4 系统设计

### 4.1 系统架构设计

本系统采用分层架构设计，自上而下分为五层：

```
┌─────────────────────────────────────────┐
│           展示层（Streamlit Web）         │
│  数据采集页面 │ 数据概览 │ 可视化 │ 关联分析 │
├─────────────────────────────────────────┤
│              业务逻辑层                   │
│  AutoCollector │ DataCleaner │ DataMining │
├─────────────────────────────────────────┤
│            数据存储层                     │
│    MySQL（分表）  │  MongoDB（文档）       │
├─────────────────────────────────────────┤
│            数据处理层                     │
│   Spark RDD聚类  │  Pandas数据清洗         │
├─────────────────────────────────────────┤
│            数据采集层                     │
│    BilibiliSpider（Requests + API）       │
```

**系统架构说明：**
- 数据采集层：通过Requests库调用B站API，获取视频数据
- 数据处理层：使用Pandas进行数据清洗，使用Spark RDD进行分布式聚类分析
- 数据存储层：MySQL按日期分表存储结构化数据，MongoDB存储文档备份
- 业务逻辑层：AutoCollector管理定时采集，DataCleaner处理数据清洗，DataMining执行Spark分析
- 展示层：Streamlit构建Web界面，Plotly生成交互式图表

### 4.2 系统模块设计

#### 4.2.1 数据采集模块（bilibili_spider.py + auto_collector.py）

**功能描述：** 负责通过B站API搜索接口采集视频数据。

**核心组件：**
- BilibiliSpider类：封装API请求、搜索、视频详情获取等功能
- AutoCollector类：继承BilibiliSpider，管理定时自动采集和手动采集

**关键技术点：**
- 使用B站搜索API：`api.bilibili.com/x/web-interface/search/type`
- 视频详情API：`api.bilibili.com/x/web-interface/view?bvid={bvid}`
- 日期筛选：通过pubdate时间戳过滤指定日期的视频
- 延时控制：delay_between_api_requests（0.3秒基础延时）、delay_between_pages（0.5-1.5秒随机翻页延时），防止请求过快被B站限制
- 定时调度：使用后台线程（threading.Thread + daemon=True）轮询检查时间

#### 4.2.2 数据清洗与存储模块（data_cleaner.py + data_storage.py）

**功能描述：** 清洗原始数据和双库存储管理。

**核心组件：**
- DataCleaner类：数据清洗、去重、标准化
- DataStorage类：MySQL + MongoDB存储管理

**关键技术点：**
- HTML标签移除：使用正则表达式`<[^>]+>`匹配并移除
- 数字标准化：处理"万"（×10000）、"亿"（×100000000）等中文单位
- 按日期分表：表名格式`bilibili_videos_YYYYMMDD`，自动创建
- 双库同步：MySQL和MongoDB同时存储
- 默认聚类标签：初始设为"普通视频"，由Spark分析更新

#### 4.2.3 Spark数据挖掘模块（data_mining.py）

**功能描述：** 使用Spark RDD实现基于标签分类的动态聚类分析。

**核心组件：**
- DataMining类：管理Spark连接、聚类分析、统计计算

**关键技术点：**
- **标签分类库（TAG_CLASSIFICATION）：** 定义了热门标签（游戏/美食/音乐等）、中等标签（教育/职场/财经等）、冷门标签（修鞋/木工/手工等）三类，每类包含不同的关键词和评判阈值
- **默认归类：** 未匹配到任何关键词的视频默认归为"中等标签"（最平衡的评判标准）
- **评判标准差异化：**

| 标签分类 | 爆款 | 热门 | 普通 | 冷门 |
|---------|------|------|------|------|
| 热门标签 | 50万播放/2万点赞 | 10万播放/5千点赞 | 2万播放/1千点赞 | < 2万播放 |
| 中等标签 | 20万播放/1万点赞 | 5万播放/2千点赞 | 1万播放/5百点赞 | < 1万播放 |
| 冷门标签 | 10万播放/5千点赞 | 3万播放/1千点赞 | 5千播放/2百点赞 | < 5千播放 |

- **Spark RDD处理流程：** parallelize() → map(process_video) → collect()
- **数据库更新：** 遍历聚类结果，UPDATE每个分表的cluster_label和cluster_type字段
- **分析结果表：** 自动创建5张分析结果表（基础统计、TOP视频、UP主统计、聚类分布、关联规则）
- **先清后插策略：** 每次分析前先DELETE当天分析结果，避免重复数据

#### 4.2.4 可视化模块（app.py）

**功能描述：** 使用Streamlit + Plotly构建交互式Web界面。

**核心页面：**

**（2）Spark RDD聚类流程**

使用Spark RDD进行分布式处理的完整流程（data_mining.py L155-L243）：

```python
def spark_cluster_analysis(self):
    # Step 1: 读取所有分表数据
    all_videos = [所有分表的视频数据]
    
    # Step 2: 并行化处理（RDD创建）
    video_rdd = self.spark.sparkContext.parallelize(all_videos)
    
    # Step 3: 映射处理（并行执行）
    def process_video(video):
        cluster_info = self._update_video_cluster({
            "title": video["title"],
            "play": video["play"],
            "like": video["like"]
        })
        return {"id": video["id"], ...}
    
    clustered_rdd = video_rdd.map(process_video)
    
    # Step 4: 收集结果
    clustered_list = clustered_rdd.collect()
    
    # Step 5: 更新数据库
    for video in clustered_list:
        UPDATE table SET cluster_label=%s, cluster_type=%s WHERE id=%s
```

**（3）标签匹配逻辑**

视频标题匹配的完整流程：

1. 遍历TAG_CLASSIFICATION中的所有标签分类
2. 检查标题中是否包含该分类的关键词
3. 匹配到 → 使用该分类的评判标准
4. **全部未匹配 → 默认归为"中等标签"（最平衡的评判标准）**
